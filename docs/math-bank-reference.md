# Math-bank Reference — Tag Schema + Explanation Style

> เอกสารอ้างอิงสำหรับ Claude (หรือ AI อื่น) ในการช่วยงานโปรเจค **math-bank** (คลังข้อสอบคณิตศาสตร์ ม.ปลาย ไทย — PAT1, วิชาสามัญ, A-Level, ข้อสอบรายบท) ครอบคลุม 2 ส่วนหลัก: ระบบ tag บทเรียน และ style การเขียนเฉลย

## บริบทโปรเจคโดยย่อ

- เก็บข้อสอบเป็น JSON · render ผ่าน KaTeX + Sarabun font · ธีม parchment cream
- 2 ประเภทไฟล์: **ชุดข้อสอบ** (PAT1, สามัญ, A-Level — มี year + month) และ **ข้อสอบรายบท** (chap-XX — รวมข้อจากหลายแหล่งจัดตามบท)
- ทุกข้อสอบมี: `id`, `setId`, `topics`, `subTopics`, `difficulty`, `questionNumber`, `type` (`mc`/`fill`), `question`, `choices`/`correct`, `explanation`
- ปรัชญา: เฉลยต้องชัดเจน เด็กเรียนรู้ตามได้เอง · มีเหตุผลทุกขั้น · ไม่ใช่แค่บอกคำตอบ

# PART 1 — TAG SCHEMA (18 บท)

## 1.1 รายการ tag ทั้งหมด

ข้อสอบทุกข้อต้อง tag ตามบทเรียน ม.ปลาย เก็บเป็น **string ตัวเลข** `"1"` ถึง `"18"` (ห้ามใช้ integer)

| `"2"` | ตรรกศาสตร์ | ประพจน์, ค่าความจริง, สมมูล, การอ้างเหตุผล, ตัวบ่งปริมาณ |

ชื่อทางการอยู่ใน `data/manifest.json` field `topicNames` — ถ้า mismatch ให้เชื่อ manifest

## 1.2 โครงสร้าง field

```json
{
  "topics":    ["3"],        // บทหลัก: 1-2 บทสำคัญที่สุด
  "subTopics": ["17"]        // บทรอง (tool): 0 บทขึ้นไป
}
```

**`topics` (primary)** — บทที่ข้อนั้นถามเรื่องนี้ตรง ๆ · จำนวน 1-2 บท · ห้ามว่าง · ใช้ filter ใน UI
**`subTopics` (secondary tools)** — บทที่ใช้เป็นเครื่องมือ แต่ไม่ใช่หัวข้อหลัก · 0+ บท · ไม่นับใน chapter count

## 1.3 กฎสำคัญ

1. **Format string เสมอ** — `"3"` ไม่ใช่ `3`
2. **เรียง ascending ตามค่าตัวเลข** — `["3", "8"]` ✓ · `["8", "3"]` ✗
3. **Mutual exclusion** — บทเดียวอยู่ได้แค่ที่เดียว
4. **`topics` ห้ามว่าง** — ขั้นต่ำ 1 บท
5. **`subTopics` ใส่เฉพาะที่ใช้จริงและสำคัญ** — ไม่ใส่บทกลาง ๆ ที่แค่เกี่ยวข้องคร่าว ๆ
6. **Chapter sets (ข้อสอบรายบท)** — ทุกข้อใน set ของบท X ต้องมี `topics: ["X"]` เป็นค่าหลัก

## 1.4 ตัวอย่างการตัดสินใจ

**A. โจทย์ตรงไปตรงมา** — "หา ห.ร.ม. ของ 36 และ 60"
```json
{"topics": ["17"], "subTopics": []}
```

**B. โจทย์ผสมเครื่องมือ** — "ให้ `f(x) = x³ + ax² + bx - 36` โดย `a` เป็น ห.ร.ม. ของ `b` กับ 216 ... หา `f(a)`"
```json
{"topics": ["3"], "subTopics": ["17"]}
```
หัวข้อหลักคือพหุนาม/ทฤษฎีบทเศษเหลือ · ใช้ยุคลิดเป็น tool

**C. โจทย์ข้ามบทแบบสมดุล** — "เซต `A = {x | log₂ x < 3}` จงหา `A`"
```json
{"topics": ["1", "8"], "subTopics": ["3"]}
```
เซต + log สำคัญเท่ากัน · แก้อสมการเป็น tool

**D. แคลคูลัสจาก function** — "หาค่าสูงสุดของ `f(x) = sin x + cos x` บน `[0, 2π]`"
```json
{"topics": ["13"], "subTopics": ["7"]}
```

## 1.5 Decision framework

ถามตามลำดับ:
1. โจทย์ถามเรื่องอะไรเป็นหลัก? → บทนั้นคือ `topics`
2. มี 2 หัวข้อสำคัญพอ ๆ กันหรือไม่? → ใส่ทั้งคู่ใน `topics` (max 2)
3. มีบทอื่นที่ "ต้องใช้แก้" แต่ไม่ใช่หัวข้อหลัก? → ใส่ใน `subTopics`
4. ทักษะพื้นฐาน (บวก ลบ คูณ หาร แทนค่า)? → ไม่นับ
5. ลังเล? → เลือกบทที่อยู่ใน **เนื้อหาที่สอนข้อนี้** จริง ๆ

## 1.6 Common subTopic patterns

| Primary | Common subTopics | เหตุผล |
|---------|------------------|--------|
| `"3"` ระบบจำนวนจริง | `"17"` | พหุนามที่ใช้ ห.ร.ม./ค.ร.น./ยุคลิด |
| `"5"` ฟังก์ชัน | `"7"`, `"8"`, `"3"` | โดเมน/กราฟของฟังก์ชันตรีโกณ, log, ราก |
| `"7"` ตรีโกณ | `"3"`, `"5"` | สมการตรีโกณที่ต้องแก้อสมการ |
| `"9"` เวกเตอร์ | `"7"` | หามุมระหว่างเวกเตอร์ (cosine) |
| `"12"` การนับ | `"1"` | นับสมาชิกเซตด้วย inclusion-exclusion |
| `"12"` ตัวแปรสุ่ม (RV) | `"15"` | RV แปลงเชิงเส้น aX+b โยงสมบัติค่ากลาง-การกระจาย |
| `"13"` แคลคูลัส | `"5"`, `"7"`, `"8"` | อนุพันธ์/อินทิเกรตของฟังก์ชันหลายชนิด |
| `"15"` สถิติ | ส่วนใหญ่ว่าง | สถิติมักเป็นเรื่องของตัวเอง |
| `"18"` การเงิน | `"8"`, `"11"` | ทบต้น/หาจำนวนงวด → แก้ด้วย log · ค่างวด → ผลบวกอนุกรมเรขาคณิต |

# PART 2 — EXPLANATION STYLE (STANDARD v2)

`explanation` คือ **array of strings** — แต่ละ string คือ 1 บรรทัด · บรรทัดว่าง `""` ใช้คั่น section · ใช้ `<b>...</b>` ครอบหัวข้อ section

## 2.1 ลำดับ section บังคับ (8 ขั้น)

ทุกเฉลยต้องเรียงตามนี้เป๊ะ ห้ามสลับ ห้ามข้าม (ยกเว้น 🔄 ที่เป็น optional):

```
1. 📐 ความรู้พื้นฐาน         — ทฤษฎี/สูตรที่จะใช้
2. 🎯 เป้าหมาย              — โจทย์ถามอะไร ต้องทำอะไร
3. ขั้นที่ 1, 2, 3, ...      — แสดงวิธีคิดเป็นขั้น พร้อมเหตุผลนำ
4. 🔄 วิธีอื่น               — (optional) วิธีต่างมุมจริง verify ถูก
5. ✔ ตรวจคำตอบ             — แทนค่ากลับ / เช็ค edge case
6. ✅ คำตอบ                — สรุปคำตอบสุดท้าย → ตัวเลือก N
7. 💡 เทคนิคที่ใช้           — insight สำคัญที่ใช้แก้
8. ⚠️ จุดที่เด็กมักผิด        — กับดักของโจทย์, ความเข้าใจผิดทั่วไป
```

แต่ละ section คั่นด้วย `""` (empty string) ใน array

## 2.2 รายละเอียดแต่ละ section

### 📐 ความรู้พื้นฐาน
- ระบุ **ทฤษฎี/สูตร/นิยาม** ที่จะใช้แก้ข้อนี้ (1-3 บรรทัด)
- เขียนเป็น "ความรู้ทั่วไป" ไม่ใช่การประยุกต์เฉพาะข้อนี้
- ใช้ LaTeX กรอบ `$...$` สำหรับสูตร

ตัวอย่าง:
```
<b>📐 ความรู้พื้นฐาน</b>
ทฤษฎีบทเศษเหลือ: เศษจากการหาร $f(x)$ ด้วย $x - a$ คือ $f(a)$ ; และขั้นตอนวิธียุคลิดหา ห.ร.ม.
```

### 🎯 เป้าหมาย
- บรรทัดเดียว · บอกชัด ๆ ว่าจะทำอะไรเพื่อให้ได้คำตอบ
- ขึ้นต้นด้วย `<b>🎯 เป้าหมาย</b>` (ไม่ใส่ colon ก็ได้) เว้นวรรคแล้วต่อความ

ตัวอย่าง:
```
<b>🎯 เป้าหมาย</b> หาค่า $a$ และ $b$ ก่อน แล้วคำนวณ $f(a)$
```

### ขั้นที่ N
- ขึ้นต้น `<b>ขั้นที่ 1:</b>`, `<b>ขั้นที่ 2:</b>` ... (ใส่ colon)
- ทุกขั้นต้องมี **"เหตุผลนำ"** ก่อนการคำนวณ (เพราะอะไร ทำไม)
- ใช้ `⟹` (U+27F9) แทน "ดังนั้น" หรือ "implies" ภายในสมการเชิงตรรกะ
- ใช้ `→` (U+2192) สำหรับการแปลงรูป

ตัวอย่าง:
```
<b>ขั้นที่ 1:</b> จาก $216 = bq_1 + 106$ และสมบัติยุคลิด ห.ร.ม.$(216, b) = $ ห.ร.ม.$(b, 106)$
<b>ขั้นที่ 2:</b> ต่อยุคลิด — $106 = 4(26) + 2$ , $4 = 2(2) + 0$ ⟹ ห.ร.ม.$(106, 4) = 2$
```

### 🔄 วิธีอื่น (Optional — ใส่เฉพาะเมื่อมีคุณภาพ)
- **กฎ:** ต้องเป็น (1) วิธีต่างมุมจริง ไม่ใช่ restatement (2) ถูกต้องทางคณิตศาสตร์ verify ได้ (3) **inferior ไม่เอา** — ต้องเร็ว/สะอาด/insight มากกว่าหรือเสมอวิธีหลัก
- ขึ้นต้น `<b>🔄 วิธีอื่น:</b>`
- ห้ามใส่เพื่อให้ครบ section · ถ้าไม่มีวิธีอื่นที่ดี **ข้ามไป**

ตัวอย่าง:
```
<b>🔄 วิธีอื่น:</b> คิดรวบเป็นอัตราส่วนเดียว $\dfrac{y_2}{y_1} = \sqrt{\dfrac{p_2}{p_1}} = \sqrt{\dfrac{16}{9}} = \dfrac{4}{3}$
```

### ✔ ตรวจคำตอบ
- แทนคำตอบกลับเข้าโจทย์ หรือเช็คเงื่อนไข
- ใช้ `✓` (U+2713) เป็นเครื่องหมาย confirm
- 1-3 บรรทัด

ตัวอย่าง:
```
<b>✔ ตรวจคำตอบ:</b> $\dfrac{1}{3} + \dfrac{1}{6} = \dfrac{2}{6} + \dfrac{1}{6} = \dfrac{3}{6} = \dfrac{1}{2}$ ✓ ตรงกับเวลารวม $2$ ชั่วโมง
```

### ✅ คำตอบ
- รูปแบบบังคับ **mc:** `<b>✅ คำตอบ: <คำตอบ> → ตัวเลือก N</b>` (N = 1-based)
- รูปแบบบังคับ **fill:** `<b>✅ คำตอบ: <ค่า></b>`

ตัวอย่าง:
```
<b>✅ คำตอบ: $200$ → ตัวเลือก 2</b>
<b>✅ คำตอบ: $210$</b>
```

### 💡 เทคนิคที่ใช้
- บอก **insight** หรือ **shortcut** ที่ใช้ — สิ่งที่ทำให้ข้อนี้แก้ได้เร็ว/ฉลาด
- ไม่ใช่แค่ summary วิธี · ต้องเป็น "key trick"

ตัวอย่าง:
```
<b>💡 เทคนิคที่ใช้:</b> ทฤษฎีบทเศษเหลือ (เศษ $= f(a)$) ทำให้ไม่ต้องหารยาว
```

### ⚠️ จุดที่เด็กมักผิด
- **กับดักของโจทย์** หรือ **ความเข้าใจผิดทั่วไป**
- ระบุเฉพาะเจาะจง · อ้างถึงตัวเลือกผิด (distractor) ได้ถ้ามี

ตัวอย่าง:
```
<b>⚠️ จุดที่เด็กมักผิด:</b> ตอบ $6$ ชั่วโมง (เครื่องช้า) ทั้งที่ตัวเลือกมีแต่ $3$ ; เผลอเลือกราก $t = -2$ ; ตัวเลือก $2 + \dfrac{\sqrt{10}}{2}$ เป็นกับดักจากการแก้สมการผิด
```

## 2.3 ตัวอย่างเฉลยเต็มรูปแบบ (JSON array)

```json
"explanation": [
  "<b>📐 ความรู้พื้นฐาน</b>",
  "อัตราการทำงานรวมกัน: ถ้าเครื่องหนึ่งทำเสร็จใน $t$ ชั่วโมง อัตราคือ $\\dfrac{1}{t}$ ของถังต่อชั่วโมง ; เปิดพร้อมกันให้อัตรารวมกัน",
  "",
  "<b>🎯 เป้าหมาย</b> ตั้งสมการเวลาแล้วแก้หาเวลาของแต่ละเครื่อง",
  "",
  "<b>ขั้นที่ 1:</b> ให้เครื่องเร็วใช้เวลา $t$ ชั่วโมง เครื่องช้าใช้เวลา $t + 3$ ชั่วโมง",
  "",
  "<b>ขั้นที่ 2:</b> เปิดพร้อมกันเต็มถังใน $2$ ชั่วโมง ⟹ $\\dfrac{1}{t} + \\dfrac{1}{t+3} = \\dfrac{1}{2}$",
  "",
  "<b>ขั้นที่ 3:</b> คูณทั้งสองข้างด้วย $2t(t+3)$ — $2(t+3) + 2t = t(t+3)$ ⟹ $4t + 6 = t^2 + 3t$",
  "",
  "<b>ขั้นที่ 4:</b> จัดรูป $t^2 - t - 6 = 0$ ⟹ $(t - 3)(t + 2) = 0$ ⟹ $t = 3$ หรือ $t = -2$",
  "",
```

# PART 3 — TECHNICAL RULES (LaTeX/HTML/POST-FLIGHT)

## 3.1 LaTeX/KaTeX

ทุกสูตรคณิตศาสตร์ต้องครอบด้วย `$...$` (inline) — display math `$$...$$` ใช้น้อยมาก

### Commands ที่ใช้ได้แน่นอน (KaTeX supported)

```
เศษส่วน:        \dfrac{a}{b}   (display-style, ตัวใหญ่)  หรือ \frac{a}{b}
ราก:           \sqrt{x}        \sqrt[n]{x}
ดัชนี:         x^{2}            x_{1}             x^{a+b}
operators:     \times  \cdot  \pm  \mp  \div
comparison:    \le  \ge  \ne  \approx  \equiv
infinity:      \infty
sets:          \cup  \cap  \in  \notin  \subset  \subseteq  \emptyset
brackets:      \left[ \right]   \left( \right)   \left\{ \right\}
ellipsis:      \ldots  \cdots
arrows:        \to  \rightarrow  \Rightarrow  \implies
spacing:       \,  (thin)   \;  (medium)
ตัวห้อย:        \log_{2}  \log_{10}
trig:          \sin  \cos  \tan  \sec  \csc  \cot  \sin^{-1}
จัดรูป:        \overline{x}  \bar{x}   \vec{v}   \hat{n}
sum/int:       \sum_{i=1}^{n}   \int_{a}^{b}
matrix:        \begin{bmatrix} a & b \\ c & d \end{bmatrix}
```

### Symbols Unicode ที่ใช้ได้ตรง ๆ (ไม่ต้องอยู่ใน $...$)

```
⟹  (U+27F9)  — "implies" ใช้แทน "ดังนั้น" ในเชิงตรรกะ
→  (U+2192)  — ลูกศรเปลี่ยนรูป
✓  (U+2713)  — เครื่องหมายตรวจถูก
≤ ≥ ≠ ± ∞ ∈ ∉ ∪ ∩  — ใช้นอก $...$ ได้ตามต้องการ (แต่ใน math mode ใช้ \le, \ge ฯลฯ)
```

## 3.2 ห้ามทำ (Common Gotchas)

### ❌ ห้าม bare `√` (U+221A)
```
ผิด:  จำนวนต้องเป็น √2 มากกว่า √3
ถูก:  จำนวนต้องเป็น $\sqrt{2}$ มากกว่า $\sqrt{3}$
```
**เหตุผล:** Unicode √ เดี่ยว ๆ ไม่มี vinculum (เส้นขีดบน radicand) อ่านไม่รู้ว่าครอบอะไรอยู่

### ❌ ห้าม `\n` literal (backslash-n)
ใน Python source code ใช้ `\n` (real newline char) สำหรับ multi-line strings · **ห้าม** เขียน `\\n` ที่ JSON.dumps จะเก็บเป็น 2-char literal
```
ผิด:  "บรรทัด 1\\nบรรทัด 2"     # 2 chars \n ใน JSON file
ถูก:  "บรรทัด 1\nบรรทัด 2"      # real newline char
```
POST-FLIGHT regex: `\\n(?![a-zA-Z])` จะจับ literal `\n` ที่ไม่ใช่ command อย่าง `\ne`, `\notin`

### ❌ ห้าม backslash-quote (`\"`) ใน explanation
ใช้ double-quote ตรง ๆ ใน Python source — `json.dumps` จะ escape เป็น `\"` ในไฟล์เอง (ถูกต้อง)
```
ผิด:  "ตัวเลือก \\\"ก. ถูก\\\" ตรงกับ..."   # backslash โผล่ใน HTML innerHTML
ถูก:  'ตัวเลือก "ก. ถูก" ตรงกับ...'         # json.dumps จัดการเอง
```

### ❌ ห้าม `<` / `>` ติดตัวอักษรในคณิต
```
ผิด:  $y<x$  →  browser parse เป็น HTML tag <x>
ถูก:  $y < x$  (เว้นวรรครอบ < และ >)
```
ทั่วไป: ติดตัวเลขปลอดภัย (`$3<5$`) · ติดตัวอักษรอันตราย (`$y<x$`) · ไม่เสียอะไรเว้นวรรคทุกที่

### ❌ ห้าม `\text{}` ครอบภาษาไทย
KaTeX render Thai font ไม่ดีใน `\text{}` · ถ้าต้องการคำไทยในสูตร ให้ออกจาก math mode
```
ผิด:  $\dfrac{y_{\text{อนาคต}}}{y_{\text{ปัจจุบัน}}}$
ถูก:  อัตราส่วน $\dfrac{y_2}{y_1}$ เมื่อ $y_2$ คืออนาคต $y_1$ คือปัจจุบัน
```

### ⚠️ `\ne` ใน preview render เพี้ยน
ใน local preview HTML (KaTeX) `\ne` บางครั้ง render เป็น `≡` — **เป็น font fallback bug ของ KaTeX** ไม่ต้องแก้ JSON · admin viewer จริง render ถูก

## 3.3 Field order ใน JSON (ห้ามสลับ)

**MC:**
```
id, setId, topics, subTopics, difficulty, questionNumber, type,
question, choices, correct, explanation
```

**Fill:**
```
id, setId, topics, subTopics, difficulty, questionNumber, type,
question, correct, accept, explanation
```

- `id` pattern: `<setId>-q<N>` (เช่น `chap-03-realnumber-q97`, **ไม่ zero-pad**)
- `correct` ใน mc = 0-based index · ใน fill = string ของคำตอบ
- `accept` ใน fill = array รูปแบบที่ยอมรับ (เช่น `["210"]` หรือ `["1/2", "0.5"]`)

## 3.4 File format

- **Encoding:** UTF-8 (no BOM)
- **Line endings:** LF (Unix), no CRLF
- **No trailing newline**
- **Indent:** ส่วนใหญ่ `indent=2` (เช็ค manifest + ไฟล์เดิมก่อนเขียนเสมอ)
- **Python write:** `io.open(path, 'w', encoding='utf-8', newline='\n').write(json.dumps(d, ensure_ascii=False, indent=2))`

## 3.5 POST-FLIGHT checks (รันก่อน save ทุกครั้ง)

```python
import json, io, re

data = json.load(io.open(filepath, encoding='utf-8'))
qs = data['questions']
KEY = {...}  # mc: question_number → 1-based correct (จาก answer key PDF)

sig = re.compile(r'\\(begin|frac|dfrac|sqrt|cdot|le|ge|ne|pm|infty|vec|left|right|overline|sum|int|forall|exists|sim|cup|cap|notin|times|ldots|approx)')
bad_nl = re.compile(r'\\n(?![a-zA-Z])')

issues = 0
for q in qs:
    n = q['questionNumber']
    # (1) choices หลุด $
    for i, c in enumerate(q.get('choices', [])):
        if sig.search(c) and '$' not in c:
            print(f'หลุด$ Q{n}[{i}]'); issues += 1
    # (2) \n literal
    for f in [q.get('question','')] + q.get('choices',[]) + q.get('explanation',[]):
        if bad_nl.search(f): print(f'\\n literal Q{n}'); issues += 1
    # (3) bmatrix ดิบนอก $
    for line in q.get('explanation', []):
        if '\\begin{bmatrix' in line and '$' not in line:
            print(f'bmatrix Q{n}'); issues += 1
    # (4) คำยอมแพ้
    for line in q.get('explanation', []):
        for kw in ['ยอมแพ้', 'ตามเฉลย', 'ไม่ตรง']:
            if kw in line: print(f'ยอมแพ้ Q{n}'); issues += 1
    # (5) bare √
    if '\u221a' in json.dumps(q, ensure_ascii=False):
        print(f'bare-√ Q{n}'); issues += 1
    # (6) correct ตรง key
    if n in KEY and q['type'] == 'mc':
        if q.get('correct') is None or q['correct']+1 != KEY[n]:
            print(f'correct ผิด Q{n}'); issues += 1
    # (7) balanced $
    for f in [q.get('question','')] + q.get('choices',[]) + q.get('explanation',[]):
        if f.count('$') % 2 != 0:
            print(f'$ ไม่สมดุล Q{n}'); issues += 1

# (8) UTF-8 decode สะอาด
try: io.open(filepath,'rb').read().decode('utf-8')
except Exception as e: print('decode error', e); issues += 1

print('TOTAL ISSUES:', issues)
# ต้อง = 0 ก่อน save
```
